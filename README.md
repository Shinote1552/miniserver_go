# URL Shortener Service

Мини-сервер на Go для сокращения URL с интеграцией in-memory и PostgreSQL хранилищ. Проект полностью контейнеризован с использованием Docker.


## Архитектура и структура проекта

Проект реализован с четким разделением ответственности по слоям:

- **Domain Models**: `internal/domain/models/` - чистые бизнес-сущности, не зависящие от других слоев
- **Services**: `internal/services/` - бизнес-логика (use cases), работающая исключительно с доменными моделями
- **Handlers**: `internal/http/handlers/` - HTTP контроллеры
- **DTOs**:
  - `internal/http/dto/` - транспортные модели для HTTP слоя
  - `internal/repository/dto/` - модели репозитория для работы с БД
- **Repository** - слой работы с данными с реализацией интерфейсов, определенных в сервисном слое

**Преимущества архитектуры**:
- Интерфейсы как четкие контракты между слоями
- Изолированное тестирование компонентов с mock заглушками
- Возможность замены реализаций без изменения бизнес-логики
- Независимость доменного слоя от инфраструктурных деталей

## Конфигурация системы

### Приоритеты настроек (от высшего к низшему)
1. Флаги командной строки
2. Переменные окружения
3. Значения по умолчанию (хардкод в `config` пакете)


### Доступные флаги командной строки

| Флаг                  | Описание                          | Пример значения                  |
|-----------------------|-----------------------------------|----------------------------------|
| `-server-address`     | Адрес и порт сервера              | `-server-address=":8080"`        |
| `-base-url`           | Базовый URL для ссылок            | `-base-url="https://example.com"`|
| `-file-storage-path`  | Путь к файлу хранилища URL        | `-file-storage-path="/data/urls.json"`|
| `-database-dsn`       | DSN для подключения к PostgreSQL  | `-database-dsn="postgres://user:pass@localhost:5432/db"`|
| `-jwt-access-expire`  | Время жизни Access токена         | `-jwt-access-expire="15m"`       |

### Переменные окружения

| Переменная              | Описание                          | Пример значения                          |
|-------------------------|-----------------------------------|------------------------------------------|
| `SERVER_ADDRESS`        | Адрес сервера                    | `:8080` или `0.0.0.0:9000`              |
| `BASE_URL`              | Базовый URL для ссылок           | `https://short.example.com`              |
| `FILE_STORAGE_PATH`     | Путь к файлу хранилища           | `/var/data/urls.json`                    |
| `DATABASE_DSN`          | PostgreSQL DSN строка            | `postgres://user:pass@host:5432/dbname`  |
| `JWT_SECRET_KEY`        | Секретный ключ для JWT           | `base64-encoded-32-byte-key`             |
| `JWT_ACCESS_EXPIRE`     | Время жизни Access токена        | `15m` (15 минут)                         |

## Основные возможности
### Особенности реализации
- Автоматическая JWT-аутентификация через куки и Secure Flag

- Две реализации хранилищ: PostgreSQL и in-memory, пока что полноценный Fallback не реализован

- Сжатие ответов (gzip) по хэдеру "Accept-Encoding: gzip"

- Логирование запросов через middleware

- InMemory хранилище с RWMutex для потокобезопасного доступа

- Connection Pool для работы с PostgreSQL

- TxManager для управления транзакциями с разными уровнями изоляции

- Единый интерфейс Querier для работы с БД и транзакциями

- Subrouter для разделения публичных и защищенных маршрутов

- Создание и валидация JWT токенов (HMAC-SHA256)

## API endpoints

### Публичные обработчики
`GET /ping`
Проверяет соединение с базой данных

Возвращает 200 OK если БД доступна, 500 если нет

Используется для мониторинга работоспособности

`GET /{id}`
Перенаправляет на оригинальный URL по короткому идентификатору

Возвращает HTTP 307 (Temporary Redirect)

Если URL не найден - возвращает 400 Bad Request

`GET /`
Дефолтный обработчик для корневого пути

Всегда возвращает 400 Bad Request

Используется для обработки некорректных запросов

### Защищенные обработчики (требуют JWT)
- `POST /api/shorten`
Создает короткую версию URL из JSON-запроса

Возвращает 201 Created с коротким URL

При дубликате URL возвращает 409 Conflic

Валидирует входные данные

- `POST /api/shorten/batch`
* * Пакетное создание коротких URL

* * Принимает массив URL с correlation_id

* * Возвращает массив результатов с сохранением correlation_id

* * Обрабатывает до 10 URL за один запрос

`POST /`
* * Альтернативная версия сокращения URL

* * Принимает URL в теле запроса (text/plain)

* * Возвращает короткий URL как текст

* * Аналогичная логика обработки дубликатов

`GET /api/user/urls`
* * Возвращает все URL текущего пользователя

* * Формат: массив объектов {short_url, original_url}

* * Пустой результат - 204 No Content

* * Требует валидной JWT куки

## Проблемы реализации и TODO
### Текущие ограничения
- Логирование реализовано только на уровне middleware, отсутствует структурированное логирование на других слоях

- Отсутствует rate limiting - нет ограничений на количество запросов

- Базовая обработка ошибок - можно улучшить детализацию и формат ошибок


### Планируемые улучшения
- SingleFlight - для предотвращения дублирующих запросов при конкурентном доступе

- ErrGroup - для управления группами горутин и обработки ошибок

- Структурированное логирование

## Тестовые сценарии
### Проект включает несколько тестовых окружений:

* Основное (PostgreSQL):

* * Порт: 8080

* * Надежное персистентное хранилище

* * Используется для production-like тестирования

*

* InMemory (для тестирования):

* * Порт: 8081

* * Быстрое тестирование без зависимостей

* * Имитация работы без БД

*

* Fallback (при недоступности PostgreSQL):

* * Порт: 8082

* * Тестирование поведения при недоступности БД

* * Проверка автоматического перехода на inmemory
